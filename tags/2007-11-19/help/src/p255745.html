Facilitates reporting on aborted checkverify and checkstorage operations.

sp_dbcc_patch_finishtime dbname, opid [, optype [, seq [, finishtime]]]

Parameter information :
dbname : The name of the database checkstorage or checkverify was working on when it aborted.
opid : The operation id corresponding to the aborted operation. See the errormessage for the value to use.
optype : Type of operation that was aborted. Possible values: checkstorage, checkverify
seq : The checkverify sequence number, see the given errormessage for the value to use. Use null for checkstorage.
finishtime : Set the value for the incompleted checkstorage or checkverify operation. Use null for current date and time.
