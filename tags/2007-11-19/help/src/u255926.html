The executable form of the ASE program.

sqlsrvr [-f] [-g] [-G] [-h] [-H] [-m] [-P] [-q] [-v] [-X] [-a path_to_CAPs_directive_file] [-b master_device_size] [-c config_file_for_server] [-d device_name] [-e path_to_error_log] [-i interfaces_file_directory] [-K keytab_file] [-L config_file_name_for_connectivity] [-M shared_memory_repository_directory] [-p sa_login_name] [-r mirror_disk_name] [-s server_name] [-T trace_flag] [-u sa_sso_name] [-w {master | model }] [-y [password] ] [-z page_size [ k | K ] ]

Parameter information :
-f : forces initialization of a device or database. You must use both -b and -w to use -f.
-g : turns off event-logging.
-G : specifies the name of the event log server.
-h : Just print a help message.
-H : starts the High Availability (HA) server, if you have the HA feature installed on your ASE.
-m : starts ASE in single-user mode.
-P : Run at a high priority.
-q : treats quiesced databases as 'in recovery'.
-v : Print a version number.
-X : starts this server as sybmon, not dataserver.
-a : specifies the path to the CAPs directive file.
-b : specifies the size of the master device.
-c : specifies the full path name of an ASE configuration file.
-d : is the full path name of the device for the master database.
-e : is the full path name of the error log file for ASE system-level error messages.
-i : specifies the directory location of the interfaces file to search when connecting ASE.
-K : specifies the path to the keytab file used for authentication in DCE.
-L : specifies the name the configuration file for connectivity.
-M : places shared memory files in the specified directory instead of in the default location %SYBASE%.
-p : specifies the login name of a System Security Officer when starting ASE, for the purposes of getting a new password for that account.
-r : starts the mirror of the master device.
-s : specifies the name of the ASE to start.
-T : Trace flags
-u : specifies the System Administrator or System Security Officer's name you want to unlock.
-w : specifies whether you want to write a master or model database.
-y : allows you to assign a password for the encrypted private key, so that the server prompt the user for a password.
-z : specifies the page size of the server.
