Dynamically starts and stops listeners on any given port on a per-engine basis.

sp_listener cmd, addr, {eng | remaining}

Parameter information :
cmd : The function you would like to perform. Possible values: "resume" : instructs suspended listeners to resume listening again, "start" : starts a listener on the specified ports on each of the specified engines, "status" : report on the state of the listeners specified by the parameters, "stop" : terminates the specified listeners, "suspend" : prevent the listener from accepting any more connections
addr : Can be the name of a server as specified in the interfaces file or an address in the form of '[protocol:]machine:port'. Protocol can be tcp, tli, ssltcp, ssltli, winsock, sslnlwnsck, sslwinsock.
eng : specifies the number of the engine affected by this command (this parameter is ignored by Windows NT. engine can be a single-engine number in quotes ('2'), a list ('3,5,6'), a range ('2-5'), or mix of all ('2,3-5,7')
remaining : specifies that the command is to take effect on all engines on which it can be meaningfully applied (that is, where the listener is in a state in which the command is can take effect).
