The executable form of the Backup Server program, located in $SYBASE/ASE-12_5/bin.

backupserver [-C server_connections] [-S b_servername] [-I interfaces_file] [-e error_log_file] [-M sybmultbuf_binary] [-N network_connections] [-T trace_value] [-L Sybase_language_name] [-J Sybase_character_set_name] [-c tape_config_file] [-D bitmap_number] [-A pathname] [-P active_service_threads] [-V level_number] [-p packet_size] [-m max_shared_memory] [-v]

Parameter information :
-C : Specifies the number of server connections for the Backup Server.
-S : Specifies the name of the Backup Server to start. The default is SYB_BACKUP.
-I : Specifies the name and location of the interfaces file to search when connecting to Backup Server.
-e : Specifies the name and location of the Backup Server error log file used to report Open Server internal errors, sybmultbuf errors, errors that halt the Backup Server, and errors for disconnected sessions.
-M : Specifies the full path name of the sybmultbuf executable.
-N : Specifies the number of total network connections that the master Backup Server can originate.
-T : Enter a trace value here, interpreted as a bitmask (base-2 number).
-L : Specifies the default language for Backup Server.
-J : Specifies the default character set for Backup Server.
-c : Specifies the name and location of the tape configuration file to search for tape device configuration information before doing a dump database or a dump transaction.
-D : Specifies the bitmap (base 10 number) of the diagnostic flags used within Backup Server.
-A : Specifies the pathname to the directory of the Archive API dynamically loadable library.
-P : Allows you to increase the number of stripes during multiple dump/load operations
-V : Limits the messages that are printed to the Backup Server error log.
level_number : The level_number variable determines the degree of error verbosity for Backup Server. Possible values: 0 : Displays all messages, including backup progress, 1 : All -V2 messages plus phase messages, 2 : Displays all -V3 messages plus file creation and file mount messages., 3 : displays only completion messages from a normal dump or load command and error messages from Backup Server and sybmultbuf, volume change messages, Open Server messages, Trace print messages, Informational messages from the System & Tape Auto Config module
-p : Specifies the TDS packet size in bytes that the local Backup Server requests from the remote Backup Server during network dumps.
-m : Specifies the maximum amount of shared memory in megabytes that Backup Server can use for all of its dump or load sessions.
-v : Prints the version number
