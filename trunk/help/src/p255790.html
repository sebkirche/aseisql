Calculates the space requirements for a table or index, and the time it takes to create an index.

sp_estspace table_name, no_of_rows [, fill_factor [, cols_to_max [, textbin_len [, iosec [,pagesize]]]]]

Parameter information :
table_name : Name of the table.
no_of_rows : Estimated number of rows for the table.
fill_factor : Fillfactor for the index.
cols_to_max : A comma separated list of the variable length columns for which the maximum length should be used in the calculation.
textbin_len : The length per row of all text and image columns.
iosec : Number of disk io's per second.
pagesize : allows you to estimate the space required for a given table and all of its indexes if you migrate the table to a server of the specified page size. You can either specify a page size (2, 4, 8, or 16K) or NULL to use your current page size. Because page allocation allocates the same size page for various objects, the page_size value applies to all page types (index, data, text and so on).
