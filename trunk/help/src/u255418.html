The executable form of the ASE program.

dataserver [-f] [-g] [-G] [-h] [-H] [-m] [-q] [-X] [-a path_to_CAPs_directive_file] [-b master_device_size [k | K | m | M | g | G]] [-c config_file_for_server] [-d device_name] [-e path_to_error_log] [-i interfaces_file_directory] [-K keytab_file] [-L config_file_name_for_connectivity] [-M shared_memory_repository_directory] [-p sa_login_name] [-r mirror_disk_name] [-s server_name] [-T trace_flag] [-u {sa_name | sso_name}] [-w {master | model}] [-y [password]] [-z page_size [ k | K ]] [-v]

Parameter information :
-f : forces initialization of a device or database. -f is valid only when used with -b and/or -w. The server fails to boot if you use -f without either -b or -w. -f forces the server in different ways, depending whether -w is present.
-g : turns off event-logging.
-G : specifies the name of the event log server.
-h : Print a list of the possible options and some other help.
-H : starts the High Availability (HA) server, if you have the HA feature installed on your ASE.
-m : Sarts ASE in single-user mode.
-q : treats quiesced databases as 'in recovery'.
-X : starts this server as sybmon, not dataserver.
-a : specifies the path to the CAPs directive file.
-b : specifies the size of the master device or database you want to build. The server calculates the sizes, so you can use 'K', 'M', and 'G' instead of exact byte numbers.
-c : specifies the full path name of an ASE configuration file. Use this parameter to start ASE with the configuration values in the specified configuration file.
-d : is the full path name of the device for the master database. The master database device must be writable by the user who starts ASE. If you do not use the -d parameter, the default master database device name is d_master.
-e : is the full path name of the error log file for ASE system-level error messages.
-i : specifies the directory location of the interfaces file to search when connecting ASE. If -I is omitted, dataserver looks for a file named interfaces in the directory pointed to by your SYBASE environment variable.
-K : specifies the path to the keytab file used for authentication in DCE.
-L : specifies the name the configuration file for connectivity.
-M : places shared memory files in the specified directory instead of in the default location, $SYBASE.
-p : specifies the login name of a System Security Officer when starting ASE, for the purposes of getting a new password for that account.
-r : starts the mirror of the master device. Use this parameter to start ASE if the master device has been damaged.
-s : specifies the name of the ASE to start. If -s is omitted, a server named SYBASE is started.
-T : Traceflag value
-u : specifies the System Administrator or System Security Officer's name you want to unlock.
-w : specifies whether you want to write a master or model database.
-y : allows you to assign a password for the encrypted private key, so that the server prompts the user for a password.
-z : specifies the page size of the server. You must use -b and -w to use this flag, and name an even power of two between 2k and 16k, or else the server does not boot.
-v : prints the version number and copyright message for dataserver, then exits.
