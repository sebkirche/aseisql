Copies definitions for specified views, rules, defaults, triggers, or procedures from a database to an operating-system file or from an operating-system file to a database.

defncopy [-X] [-a display_charset] [-I interfaces_file] [-J [client_charset]] [-K keytab_file] [-P password] [-R remote_server_principal] [-S [server_name]] [-U username] [-V security_options] [-Z security_mechanism] [-z language] { in file_name database_name | out file_name database_name [owner.]object_name [[owner.]object_name]... } [-v]

Parameter information :
-X : initiates the login with client-side password encryption in this connection to the server.
-a : runs defncopy from a terminal whose character set differs from that of the machine on which defncopy is running.
-I : specifies the name and location of the interfaces file to search when connecting to ASE.
-J : specifies the character set to use on the client.
-K : specifies the path to the keytab file used for authentication in DCE.
-P : specifies your password.
-R : specifies the principal name for the server.
-S : specifies the name of the ASE to which to connect.
-U : specifies a login name.
-V : specifies network-based user authentication.
security_options : option to enable additional security services. Possible values: c : Enable data confidentiality service, i : Enable data integrity service, m : Enable mutual authentication for connection establishment, o : Enable data origin stamping service, q : Enable out-of-sequence detection, r : Enable data replay detection
-Z : specifies the name of a security mechanism to use on the connection.
-z : is the official name of an alternate language that the server uses to display defncopy prompts and messages.
in : copy into ASE
file_name : specifies the name of the operating system file destination or source for the definition copy.
database_name : specifies the name of the database to copy the definitions from or to.
out : Copy out of ASE
file_name : specifies the name of the operating system file destination or source for the definition copy.
database_name : specifies the name of the database to copy the definitions from or to.
owner : is optional if you or the Database Owner own the table being copied.
object_name : specifies name(s) of database object(s) for defncopy to copy out. Do not use objectname when copying definitions in.
owner : is optional if you or the Database Owner own the table being copied.
object_name : specifies name(s) of database object(s) for defncopy to copy out. Do not use objectname when copying definitions in.
-v : Print version number
