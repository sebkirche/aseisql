Sample during an interval

sp_sysmon interval_or_option [ ,section] [ ,applmon] [ ,filter] [, dumpcounters] [, option]

Parameter information :
interval_or_option : The time period to sample over in format "hh:mm:ss".
section : Reporting section or null for all sections. Possible values: appmgmt : Application management, cache wizard, dcache : Data cache management, diskio : Disk I/O management, esp : Extended stored procedure management., housekeeper, indexmgmt : Index management, kernel : Kernel utilisation, locks : Lock management, mdcache : Metadata cache management., memory : Memory management, monaccess : Monitor access, netio : Network I/O, noclear : Do not clear the internal counters so that multiple sessions may run sp_sysmon (or other monitoring tools) at the same time., parallel : Parallel query management, pcache : Procedure cache management, recovery : Recovery management, repagent, taskmgmt : Task management, wpm : Worker process management, xactmgmt : Transaction management, xactsum : Transaction profile
applmon : Details per application or login or not detailed to that level. Possible values: appl_and_login : CPU, I/O, priority changes, resource limits per application name and login anme, appl_only : CPU, I/O, priority changes, resource limits per application_name, noclear : Do not clear the internal counters so that multiple sessions may run sp_sysmon (or other monitoring tools) at the same time., no_appl : No detail per application / login.
filter : The name of a cache (can only be used when section = 'cache wizard'). You can also specify the noclear option here.
dumpcounters : Dump the raw monitoring values. Possible values: n : (default), noclear : Do not clear the internal counters so that multiple sessions may run sp_sysmon (or other monitoring tools) at the same time., y
option : Do not clear the internal counters so that multiple sessions may run sp_sysmon (or other monitoring tools) at the same time. Possible values: noclear, null
