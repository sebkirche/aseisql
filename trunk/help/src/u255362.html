Copies a database table to or from an operating system file in a user-specified format.

bcp [[database_name.]owner.]table_name [:slice_number] {in | out} datafile [-m maxerrors] [-f formatfile] [-e errfile] [-F firstrow] [-L lastrow] [-b batchsize] [-n] [-c] [-t field_terminator] [-r row_terminator] -U username [-P password] [-I interfaces_file] [-S server] [-a display_charset] [-z language] [-A packet_size] [-J client_charset] [-T text_or_image_size] [-E] [-g id_start_value] [-N] [-X] [-K keytab_file] [-R remote_server_principal] [-V [security_options]] [-Z security_mechanism] [-Q] [-Y] [-v]

Parameter information :
database_name : Is optional if the table being copied is in your default database or in master.
owner : Is optional if you or the Database Owner owns the table being copied.
table_name : Is the name of the database table to copy.
slice_number : Is the number of the slice of the database table to copy.
in : Indicates a copy from a file into the database table
out : Indicates a copy to a file from the database table or view.
datafile : Is the full path name of an operating system file.
-m : Is the maximum number of nonfatal errors permitted before bcp aborts the copy.
-f : The full path name of a file with stored responses from a previous use of bcp on the same table.
-e : The full path name of an error file where bcp stores any rows that it was unable to transfer from the file to the database.
-F : Is the number of the first row to copy from an input file (default is the first row).
-L : The number of the last row to copy from an input file (default is the last row).
-b : Is the number of rows per batch of data copied (the default is to copy all the rows in one batch).
-n : Performs the copy operation using native (operating system) formats.
-c : Performs the copy operation with char datatype as the default storage type of all columns in the data file.
-t : Specifies the default field terminator.
-r : Specifies the row terminator.
-U : Specifies an ASE login name.
-P : Specifies an ASE password.
-I : Specifies the name and location of the interfaces file to search when connecting to ASE.
-S : Specifies the name of the ASE to which to connect.
-a : Allows you to run bcp from a terminal where the character set differs from that of the machine on which bcp is running.
-z : Is the official name of an alternate language the server uses to display bcp prompts and messages.
-A : Specifies the network packet size to use for this bcp session.
-J : Specifies the character set to use on the client.
-T : Allows you to specify, in bytes, the maximum length of text or image data that Adaptive Server sends.
-E : Explicitly specifies the value of a table's IDENTITY column.
-g : Specifies the value of the IDENTITY column to use as a starting point for copying data in.
-N : Skips the IDENTITY column.
-X : Specifies that, in this connection to the server, the application initiates the login with client-side password encryption.
-K : Specifies the path to the keytab file used for authentication in DCE.
-R : Sspecifies the principal name for the server as defined to the security mechanism.
-V : Specifies network-based user authentication.
security_options : -V can be followed by a security_options string of key-letter options to enable additional security services. Possible values: c : Enable data confidentiality service, i : Enable data integrity service, m : Enable mutual authentication for connection establishment, o : Enable data origin stamping service, q : Enable out-of-sequence detection, r : Enable data replay detection
-Z : Specifies the name of a security mechanism to use on the connection.
-Q : Provides backward compatibility with bcp version 10.0.4 for copying operations involving nullable columns.
-Y : Specifies that character-set conversion is disabled in the server, and is instead performed by bcp on the client side when using bcp IN.
-v : Displays the version number
